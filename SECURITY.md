# Security and Privacy Policy

Community Spirit is currently a public prototype using fictional and synthetic data only.

## Do Not Publish

- Real residential community maps or floor plans.
- Resident names, phone numbers, unit numbers, access cards, vehicle plates, visitor records, or payment data.
- Camera, door access, IoT, patrol, alarm, or security-sensitive logs.
- Internal property management documents unless explicitly cleared for public release.

## Prototype Boundaries

The demo can show how community data might be structured, but it must not be treated as a production access-control, emergency, surveillance, or robot-operation system.

Any real pilot should add permission management, audit logging, data minimization, retention rules, role-based access, security review, and resident consent flows before launch.

## CACP Protocol Boundary

CACP examples are protocol examples, not authorizations for real-world action. A CACP task contract, workflow export bundle, or pilot readiness checklist in this repository is valid only for synthetic public-demo review unless a separate real pilot has explicit consent, operator approval, privacy review, audit logging, human oversight, incident handling, and retention policy.

Please report any accidental exposure of real resident data, private property data, access-control material, camera/sensor payloads, payment data, or robot-operation logs as a security/privacy issue.
